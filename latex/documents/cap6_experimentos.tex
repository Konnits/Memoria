\chapter{Experimentos}
\label{cap:experimentos}

En este capítulo se describe la metodología experimental utilizada para evaluar el modelo propuesto en la memoria. El objetivo principal es determinar si la arquitectura presentada en el Capítulo~\ref{cap:modelo-no-equiespaciado} entrega predicciones competitivas en series de tiempo reales no equiespaciadas, comparándola con baselines relevantes del estado del arte y con baselines simples que sirven como referencia inferior.

\hfill

La campaña experimental abarca los 17 datasets reales procesados de la colección, con múltiples semillas aleatorias por dataset para cuantificar la variabilidad entre ejecuciones. Se incluyen además estudios de ablación para aislar el aporte individual de cada componente clave de la arquitectura, y un análisis de costo computacional para contextualizar los beneficios del modelo en términos prácticos.

\section{Objetivos experimentales}
\label{sec:objetivos-experimentales}

Los experimentos de esta memoria persiguen seis objetivos complementarios:

\begin{enumerate}
	\item \textbf{Verificar factibilidad}: comprobar que un encoder \Transformer\ con encoding temporal continuo y token objetivo puede entrenarse de manera estable sobre series temporales reales con timestamps irregulares.
	\item \textbf{Evaluar desempeño predictivo}: medir el error de predicción del modelo propuesto mediante métricas estándar de regresión, con énfasis en MSE, RMSE y MAE, usando el conjunto de test retenido.
	\item \textbf{Comparar contra baselines fuertes e informados}: contrastar el modelo propuesto con STraTS \cite{tipirneni2022strats}, un modelo diseñado explícitamente para observaciones irregulares.
	\item \textbf{Comparar contra baselines simples}: incluir un predictor de persistencia (último valor observado) y un modelo lineal con inyección temporal, para verificar que las mejoras no se deben a la complejidad del modelo sino a sus componentes específicos.
	\item \textbf{Cuantificar el aporte de cada componente (ablaciones)}: medir por separado el efecto del encoding temporal continuo y del token objetivo mediante variantes controladas del modelo.
	\item \textbf{Evaluar escalabilidad y causalidad (Autoregresión)}: analizar el comportamiento al escalar la complejidad paramétrica y medir el beneficio empírico de un entrenamiento autoregresivo mediante transferencia de pesos.
	\item \textbf{Evaluar costo versus beneficio}: registrar el número de parámetros y el tiempo de entrenamiento de cada modelo para contextualizar las mejoras en términos de eficiencia.
\end{enumerate}

\section{Escenario experimental}
\label{sec:escenario-real}

La evaluación se realiza sobre la colección completa de 17 datasets reales procesados, almacenados como archivos \textttbreak{real\_data\_i.csv}. El benchmark fue diseñado para recorrer secuencialmente todos los datasets, entrenando y evaluando cada modelo con un pipeline homogéneo de preprocesamiento, escalado, entrenamiento y evaluación final sobre test.

\hfill

El script de benchmark (\textttbreak{benchmark\_final.py}) incorpora las siguientes características de diseño:
\begin{itemize}
	\item \textbf{Reanudabilidad}: el progreso se guarda en un archivo CSV después de cada modelo entrenado. Si el proceso se interrumpe, puede reanudarse desde el último punto completado ejecutando el mismo comando.
	\item \textbf{Evaluación separada en test}: tras el entrenamiento, cada modelo se evalúa en el split de test (15\% de los datos), completamente separado del splits de validación y entrenamiento.
	\item \textbf{Múltiples semillas}: cada combinación (dataset, modelo) se ejecuta con tres semillas distintas (42, 84, 126) para cuantificar la variabilidad.
	\item \textbf{Análisis estadístico automático}: al finalizar, se generan tests de Wilcoxon de rango con signo emparejados por dataset, intervalos de confianza bootstrap y tablas resumen con media $\pm$ desviación estándar.
\end{itemize}

\subsection{Variables, partición temporal y preprocesamiento}
\label{sec:variables-y-preproceso}

En las configuraciones de datos reales utilizadas en esta memoria, cada serie se organiza a partir de:
\begin{itemize}
	\item una columna temporal \textttbreak{timestamp},
	\item una variable observada \textttbreak{valor}, utilizada tanto como entrada como variable objetivo de predicción,
	\item una partición temporal en entrenamiento, validación y prueba de proporciones 70\%, 15\% y 15\%, respectivamente.
\end{itemize}

Antes de entrenar los modelos, los datos se ordenan por tiempo y se normalizan mediante \textttbreak{StandardScaler}. El escalado se ajusta sólo con los datos de entrenamiento y luego se aplica a validación y prueba, evitando así fuga de información desde el futuro hacia el pasado.

\hfill

Las ventanas supervisadas se construyen respetando el orden temporal. La configuración usa una historia máxima de 500 observaciones y una historia mínima de 20 observaciones, lo que permite que el modelo vea contextos de distinta longitud durante el entrenamiento. Esta decisión es consistente con el objetivo de robustecer la arquitectura frente a cambios en la densidad temporal de muestreo.

\section{Modelos comparados}
\label{sec:modelos-comparados}

En esta memoria se comparan los siguientes modelos, agrupados en cuatro categorías:

\subsection{Arquitecturas propuestas}
\begin{itemize}
	\item \textbf{Custom (Solo-Encoder, sinusoidal)}: corresponde al modelo base descrito en el Capítulo~\ref{cap:modelo-no-equiespaciado}, utilizando un Encoder unificado que asimila historia y targets simultáneamente mediante una máscara diferenciadora. Emplea el encoding temporal sinusoidal fijo.
	\item \textbf{Custom-T2V (Solo-Encoder, Time2Vec)}: variante del modelo anterior que reemplaza el encoding sinusoidal fijo por Time2Vec (Sección~\ref{sec:time2vec}) y añade el sesgo de atención temporal (Sección~\ref{sec:temporal-attention-bias}). Representa la arquitectura completa propuesta en esta memoria.
\end{itemize}

Inicialmente se consideró también una variante Encoder-Decoder que separa la historia en un encoder bidireccional y delega la predicción a un decoder con atención cruzada. Sin embargo, los experimentos preliminares revelaron que esta arquitectura rendía consistentemente igual o peor que el modelo Solo-Encoder, pese a contar con aproximadamente 2.5 veces más parámetros. Este resultado se atribuye a que el decoder procesa muy pocos tokens objetivo (1 a 4), haciendo que sus capas adicionales no aporten capacidad útil. En consecuencia, la variante Encoder-Decoder fue descartada del benchmark final.

\subsection{Baselines del estado del arte}
\begin{itemize}
	\item \textbf{STraTS adaptado}: implementación basada en STraTS \cite{tipirneni2022strats}, adaptada mediante un módulo multi-horizonte para operar en la tarea de forecasting definida en esta memoria.
\end{itemize}

Adicionalmente, se cuenta con una implementación adaptada de CoFormer \cite{wei2023coformer}. Sin embargo, dicho modelo no fue incorporado a esta comparación porque su costo de entrenamiento resultó prohibitivamente alto para ejecutar el benchmark completo con múltiples semillas y 17 datasets. Su incorporación queda como extensión directa para trabajo futuro.

\subsection{Baselines simples}
\begin{itemize}
	\item \textbf{Persistencia}: predice el último valor observado de la historia para todos los horizontes futuros. No tiene parámetros entrenables y sirve como cota inferior del desempeño.
	\item \textbf{Lineal}: modelo con una capa de encoding lineal sobre los últimos 50 valores de la historia, concatenada con el timestamp relativo del target. Tiene parámetros entrenables pero una capacidad limitada, y permite verificar si la complejidad del Transformer se justifica frente a una alternativa más sencilla.
\end{itemize}

\subsection{Variantes de ablación}
\begin{itemize}
	\item \textbf{NoTimeEncoding}: Transformer idéntico al modelo propuesto pero donde el encoding temporal continuo se reemplaza por un positional encoding ordinal estándar (0, 1, 2, \ldots). Sirve para medir el aporte específico del encoding temporal continuo.
	\item \textbf{NoTargetToken}: Transformer idéntico al modelo propuesto pero donde el embedding de flag historia/target se reemplaza por ceros. Sirve para medir el aporte del mecanismo de token objetivo.
\end{itemize}

\section{Configuración de entrenamiento}
\label{sec:configuracion-entrenamiento}

Para evaluar el desempeño ante distintas demandas de capacidad, los modelos propuestos se instanciaron bajo las siguientes configuraciones paramétricas:
\begin{itemize}
	\item \textbf{Small}: $d_{\text{model}} = 64$, 2 capas, 4 cabezas, \emph{feed-forward} 128.
	\item \textbf{Base}: $d_{\text{model}} = 128$, 4 capas, 4 cabezas, \emph{feed-forward} 256.
	\item \textbf{Large}: $d_{\text{model}} = 256$, 6 capas, 8 cabezas, \emph{feed-forward} 512, con \emph{dropout} de 0.2.
\end{itemize}

Los modelos Small y Base utilizan un \emph{dropout} de 0.1. Todos comparten una escala temporal base de 900 segundos.

\hfill

El entrenamiento utiliza el optimizador AdamW con regularización de pesos (\emph{weight decay}), un scheduler de \emph{learning rate} con calentamiento lineal seguido de decaimiento coseno (\emph{warmup + cosine decay}), y selección del mejor checkpoint según el RMSE de validación. La ejecución se realiza en GPU con precisión mixta automática.

Para abordar las diferencias en la dinámica de convergencia entre modelos de distinto tamaño, se emplean configuraciones de entrenamiento diferenciadas por escala:
\begin{itemize}
	\item \textbf{Modelos pequeños} ($<$100K parámetros): \emph{learning rate} de $10^{-3}$, 30 épocas máximas, 2 épocas de calentamiento, \emph{weight decay} de $10^{-3}$.
	\item \textbf{Modelos medianos} (100K--1M parámetros): \emph{learning rate} de $10^{-3}$, 25 épocas, 1 época de calentamiento, \emph{weight decay} de $10^{-3}$.
	\item \textbf{Modelos grandes} ($>$1M parámetros): \emph{learning rate} de $2 \times 10^{-4}$, 60 épocas, 5 épocas de calentamiento, \emph{weight decay} de $10^{-2}$, $\beta_2 = 0.98$.
\end{itemize}

La reducción del \emph{learning rate} y el aumento del calentamiento para modelos grandes responden a la necesidad de estabilizar la convergencia en redes con más capas, donde tasas de aprendizaje elevadas en las primeras iteraciones pueden destruir la inicialización de los pesos y provocar que el modelo colapse a predicciones triviales.

\hfill

El modelo lineal usa una dimensión interna de 64 y toma los últimos 50 valores de la historia como entrada. El modelo de persistencia no requiere entrenamiento.

\hfill

Cada combinación (dataset, modelo) fue evaluada bajo tres semillas aleatorias (42, 84, 126), lo que proporciona estimaciones de varianza robustas e intervalos de confianza generalizables.

\section{Estudio de Ablación Autoregresivo (Fine-Tuning)}
\label{sec:ablacion-autoregresivo}

La irregularidad estructural en el submuestreo de eventos demanda caracterizar fuertemente la influencia del enmascaramiento secuencial asimétrico en las regresiones continuas. Por consiguiente, los modelos de perfil \textbf{EncDec} finalizados se sometieron empíricamente a la inserción de una capa exploratoria de \emph{Transfer Learning} cimentada en un pronóstico Causal estricto (Fine-Tuning Autoregresivo).

Iniciando por la validación principal, se congela el cuerpo bidireccional histórico (\emph{Encoder-Freezing Phase}) durante el $50\%$ primario de las subsecuentes épocas con el fin de contener divergencias bruscas y mitigar el sesgo clásico de \emph{Catastrophic Forgetting}. Traspuesto este filtro temporal, se reactiva con sutileza la propagación cruzada disminuyendo contundentemente al 10\% el \emph{Learning Rate} End-To-End.

Esta dinámica regenerativa ha sido evaluada de manera cruzada empleando las tres metodologías siguientes de provisión para secuencias de \emph{Teacher Forcing}:
\begin{itemize}
    \item \textbf{Modo Contiguo (\texttt{Contiguous})}: inyecta sub-redes asincrónicas en un bloque ininterrumpido inalterado valiéndose del soporte subyacente general del \texttt{AutoregressiveSequenceBuilder}.
    \item \textbf{Modo Aleatorio (\texttt{Random})}: dispersa los pronósticos y promueve sesgos de interpolación emulando el marco experimental general.
    \item \textbf{Modo Mixto (\texttt{Mixed})}: estocástica purista, intercala ambos acercamientos dictando lógicas transitorias variables por lote que propulsan el desvío atípico temporal.
\end{itemize}

\section{Métricas de evaluación}
\label{sec:metricas-evaluacion-experimentos}

La evaluación se realiza mediante cuatro métricas estándar de regresión:
\begin{itemize}
	\item \textbf{MSE} (\emph{Mean Squared Error}), sensible a errores grandes y directamente alineada con la pérdida principal utilizada en entrenamiento.
	\item \textbf{RMSE} (\emph{Root Mean Squared Error}), que expresa el error en las mismas unidades de la variable predicha.
	\item \textbf{MAE} (\emph{Mean Absolute Error}), menos sensible a valores atípicos que el MSE.
	\item \textbf{MAPE} (\emph{Mean Absolute Percentage Error}), útil como referencia relativa, aunque potencialmente inestable cuando los valores reales son pequeños o cercanos a cero.
\end{itemize}

En esta memoria se prioriza la interpretación conjunta de MSE, RMSE y MAE. El MAPE se reporta igualmente, pero se discute con mayor cautela.

\section{Análisis estadístico}
\label{sec:analisis-estadistico-experimentos}

Para verificar que las diferencias observadas entre modelos no sean producto del azar, se aplican los siguientes métodos estadísticos:
\begin{itemize}
	\item \textbf{Media $\pm$ desviación estándar}: se promedian las semillas por dataset y luego se calcula media y desviación sobre los 17 datasets, lo cual entrega una medida de la dispersión del error a través de distintos escenarios de datos.
	\item \textbf{Victorias por dataset}: se cuenta en cuántos de los 17 datasets cada modelo obtiene el menor error, lo que complementa la media agregada con información del ordenamiento por caso.
	\item \textbf{Test de Wilcoxon de rango con signo}: test no paramétrico emparejado por dataset para determinar si la diferencia entre el modelo propuesto y cada baseline es estadísticamente significativa. Se aplica con hipótesis alternativa unilateral (Custom $<$ baseline).
	\item \textbf{Intervalos de confianza bootstrap}: se generan 10\,000 muestras bootstrap de la media para construir intervalos de confianza del 95\% para cada modelo.
\end{itemize}

\section{Costo computacional}
\label{sec:costo-computacional-experimentos}

Para cada modelo se registra:
\begin{itemize}
	\item el número de parámetros entrenables,
	\item el número total de parámetros,
	\item el tiempo de entrenamiento en segundos (promedio sobre todas las corridas),
	\item el número de épocas efectivamente ejecutadas antes de early stopping.
\end{itemize}

Estos datos permiten evaluar si las mejoras en error del modelo propuesto se obtienen a un costo computacional razonable respecto de las alternativas.

\section{Reproducibilidad}
\label{sec:reproducibilidad}

Todos los experimentos son reproducibles ejecutando:
\begin{verbatim}
python scripts/benchmark_final.py
\end{verbatim}
El script admite argumentos para seleccionar rangos de datasets (\textttbreak{--start-dataset}, \textttbreak{--end-dataset}), semillas (\textttbreak{--seeds}), y desactivar ablaciones (\textttbreak{--skip-ablation}) o baselines simples (\textttbreak{--skip-baselines}). Si el proceso se interrumpe, basta con volver a ejecutar el mismo comando para reanudar desde el último punto completado.

\section{Resumen metodológico}
\label{sec:resumen-metodologico}

En síntesis, la campaña experimental de esta memoria consiste en un robusto benchmark de orquestación central iterando masivamente 17 bases de datos mundiales bajo 3 ecosistemas matriciales únicos dependientes por semilla. El flujo abarca subcapas dimensionales de \emph{Scaling} parametrizado e insta un riguroso test Causal Autoregresivo derivando 3 flujos combinatorios dictados estrictamente vía inyecciones del Teacher Forcing. Este despliegue iterativo provee la estructura indispensable que no solamente garantiza imparcialidad técnica en la evaluación estadística paramétrica a la luz de los antecedentes predominantes en Time-Series modernas, sino que provee una escalabilidad arquitectónica idónea a memorias rigurosas en el espectro civil y predictivo computacional.
